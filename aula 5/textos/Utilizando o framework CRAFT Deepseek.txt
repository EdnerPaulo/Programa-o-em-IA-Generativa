2. Pensar na Solução - Modelo TensorFlow
Arquitetura do modelo híbrido:

python
# Modelo base para classificação de padrões de código
import tensorflow as tf
from transformers import TFAutoModel, AutoTokenizer

class CodeExplainerModel(tf.keras.Model):
    def __init__(self, llm_model_name='bert-base-multilingual-cased'):
        super().__init__()
        # Codificador de código (entendimento estrutural)
        self.code_encoder = TFAutoModel.from_pretrained('microsoft/codebert-base')
        
        # Decodificador para explicação (geração de texto)
        self.explanation_decoder = TFAutoModel.from_pretrained('google/mt5-small')
        
        # Camada de atenção para alinhamento código-explicação
        self.attention = tf.keras.layers.MultiHeadAttention(num_heads=8, key_dim=64)
        
        # Camada de saída com tokenização em português
        self.output_layer = tf.keras.layers.Dense(50000, activation='softmax')  # vocab size
        
    def call(self, code_tokens, context_tokens):
        # Codifica o código
        code_encoding = self.code_encoder(code_tokens).last_hidden_state
        
        # Decodifica com atenção ao código
        attention_output = self.attention(context_tokens, code_encoding)
        explanation = self.explanation_decoder(attention_output).last_hidden_state
        
        return self.output_layer(explanation)

4. Escolha do LLM Principal
Comparativo para decisão final:

Modelo	Vantagens	Desvantagens	Indicação
Gemini	Melhor em raciocínio lógico, suporte a código	API paga, dependência Google	✅ Principal (qualidade)
Grok	Rápido, bom para código simples	Menos acurado em português	Backup para consultas simples
DeepSeek	Custo-benefício, código aberto	Menos recursos em PT-BR	Fine-tuning local
ChatGPT	Mais maduro, ótimo em explicações	Caro para escala, dependência OpenAI	Fallback premium
Blackbox	Especialista em código	Interface limitada, menos flexível	Complemento para debugging
Decisão: Gemini + DeepSeek (híbrido)

Gemini: explicações principais (alta qualidade)

DeepSeek: fallback e fine-tuning local (custo)

6. Interface com Streamlit
Arquitetura da interface:

python
import streamlit as st
import tensorflow as tf
from transformers import pipeline

# Configuração da página
st.set_page_config(page_title="CodeExplainer PT-BR", layout="wide")

# Carregar modelos (cache)
@st.cache_resource
def load_models():
    explainer = pipeline("text2text-generation", model="google/mt5-small")
    return explainer

def main():
    st.title("🧠 CodeExplainer - IA que explica código em português")
    st.markdown("Cole seu código e receba explicação linha por linha!")
    
    # Entrada do usuário
    code_input = st.text_area("📝 Seu código:", height=300, 
                              placeholder="Cole aqui seu código Python/JavaScript...")
    
    # Seleção de modelo
    model_choice = st.selectbox("Escolha a IA:", 
                                ["Gemini (recomendado)", "DeepSeek", "ChatGPT"])
    
    if st.button("🔍 Explicar Código"):
        if code_input:
            with st.spinner("Analisando código..."):
                # Pré-processamento
                cleaned_code = clean_code(code_input)
                
                # Chamada à API/Modelo
                explanation = generate_explanation(cleaned_code, model_choice)
                
                # Exibição formatada
                st.markdown("### 📖 Explicação")
                st.markdown(explanation)
                
                # Estatísticas
                st.sidebar.success(f"✅ {count_lines(code_input)} linhas analisadas")
        else:
            st.warning("Cole um código para explicar!")

if __name__ == "__main__":
    main()
